# Bug Fixes: Circuit Breaker, Leaderboard, ConcurrentlyRemoved, Cache

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix 5 confirmed bugs: circuit breaker recording success on cache hits, filtered leaderboard returning empty on cache miss, `ConcurrentlyRemoved` returning hardcoded count 0, `del_many` not clearing L1 zsets, and a misleading doc comment.

**Architecture:** Introduce `ValidationOutcome` enum on the `ContentValidator` trait so callers know whether a result was cached (skip breaker) or remote (signal breaker). Fix the leaderboard to fall back to DB. Fix the count query in the concurrent-remove edge case. Harden `del_many` against zset footgun.

**Tech Stack:** Rust, axum, sqlx, redis, moka, tokio

---

### Task 1: Add ValidationOutcome enum and update ContentValidator trait

**Files:**
- Modify: `crates/social-api/src/clients/content_client.rs`

Change `validate()` return type from `Result<bool, AppError>` to `Result<ValidationOutcome, AppError>`.

```rust
pub enum ValidationOutcome {
    /// Served from cache — no circuit breaker signal needed.
    Cached(bool),
    /// Result from a real remote call — circuit breaker must be signaled.
    Remote(bool),
}

pub trait ContentValidator: Send + Sync {
    async fn validate(&self, content_type: &str, content_id: Uuid) -> Result<ValidationOutcome, AppError>;
}
```

Update `HttpContentValidator::validate()`:
- Cache hit → `Ok(ValidationOutcome::Cached(cached == "1"))`
- HTTP success → `Ok(ValidationOutcome::Remote(true/false))`
- HTTP error → `Err(AppError::DependencyUnavailable(...))`

### Task 3: Update like_commands.rs to signal breaker only for Remote results

**Files:**
- Modify: `crates/social-api/src/services/like_commands.rs`

```rust
match self.content_validator.validate(content_type, content_id).await {
    Ok(ValidationOutcome::Remote(valid)) => {
        self.content_breaker.record_success();
        if !valid { return Err(AppError::ContentNotFound { ... }); }
        Ok(())
    }
    Ok(ValidationOutcome::Cached(valid)) => {
        // Cache hit — don't signal breaker (no HTTP call was made)
        if !valid { return Err(AppError::ContentNotFound { ... }); }
        Ok(())
    }
    Err(error) => {
        self.content_breaker.record_failure();
        Err(error)
    }
}
```

### Task 5: Fix ConcurrentlyRemoved to return real count

**Files:**
- Modify: `crates/social-api/src/repositories.rs`

In the `None` arm of `insert_like()`, query `like_counts` instead of returning 0:
```rust
None => {
    let count: i64 = sqlx::query_scalar(
        "SELECT COALESCE(total_count, 0) FROM like_counts WHERE content_type = $1 AND content_id = $2"
    )
    .bind(content_type)
    .bind(content_id)
    .fetch_optional(&mut *tx)
    .await?
    .unwrap_or(0);
    tx.commit().await?;
    Ok(InsertLikeResult::ConcurrentlyRemoved { count })
}
```
